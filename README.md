🔹 LAB5 — Консольное приложение

Реализовать консольное приложение для управления коллекцией объектов HumanBeing в интерактивном режиме.

📌 Требования
Класс коллекции должен поддерживать сортировку по умолчанию
Все ограничения полей должны быть соблюдены
Использовать коллекцию java.util.LinkedHashMap
При запуске данные загружаются из файла
Имя файла передаётся через аргументы командной строки
Формат хранения — JSON
Чтение: BufferedReader
Запись: OutputStreamWriter
Использовать Javadoc
Обработка ошибок (ввод, доступ к файлам и т.д.)
⚙️ Поддерживаемые команды
help — справка
info — информация о коллекции
show — вывод всех элементов
insert null {element} — добавить элемент
update id {element} — обновить элемент
remove_key null — удалить по ключу
clear — очистить коллекцию
save — сохранить в файл
execute_script file_name — выполнить скрипт
exit — выход без сохранения
remove_lower {element} — удалить меньшие элементы
history — последние 6 команд
replace_if_greater null {element} — заменить, если больше
average_of_impact_speed — среднее значение
max_by_real_hero — максимальный realHero
print_field_ascending_minutes_of_waiting — сортировка по времени ожидания
🧾 Формат ввода
Примитивы — в одной строке с командой
Объекты — по полям, построчно
Показывать подсказки при вводе
Для enum — выводить список значений
Обрабатывать ошибки ввода
null = пустая строка
Автогенерируемые поля не вводятся
📦 Модель данных
public class HumanBeing {
    private Long id;

    SORROW,
    LONGING,
    APATHY,
    RAGE,
    FRENZY;
}

🔹 LAB7 — База данных и многопоточность
🗄 Работа с БД
Использовать PostgreSQL
Хранение в БД вместо файла
id генерируется через sequence
Коллекция обновляется только при успешной записи
Чтение — из памяти
🔐 Пользователи
Регистрация и авторизация
Хеширование паролей (SHA-256)
Запрет действий без авторизации
Объекты привязаны к пользователю
Можно изменять только свои объекты
Логин и пароль передаются с каждым запросом
⚡ Многопоточность
Чтение запросов — Thread
Обработка — CachedThreadPool
Отправка ответов — CachedThreadPool
Синхронизация — ReadWriteLock

База: studs
